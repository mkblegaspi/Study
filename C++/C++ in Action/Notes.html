1.1.1. Objects and Scopes

  1. Global Scope
    • Class definition
    • Object definition
    • Constructor
    • Destructor
    • Output stream
    • Include
    • Main
    // Include statement
#include <iostream> // Tells the compiler to find the file iostream and include it.
//           |
//           ----> part of the standard C++ library, not part of the language.
// Class definition
  class World // A class is a type defined by a programmer.
  { // The class definition is inside the curly braces.
      public: // Means that we have nothing to hide yet (data hiding).
         World () // A Constructor is a piece of code that is to be executed everytime an object of this particular class is created.
         {           //    |
                     //    ----> Always have the same name as the class itself.
                     //    ----> May take any arguments - that's why it has a set of parentheses.
             std::cout << "Hello!" << std::endl; // ----> the object std::cout is sent the string "Hello!"
             // |                        |
             // |                        ----> end line (creates a new line).
             // ----> represents the standard output, presumably the screen of the computer, and it prints whatever is sent to it.
         }
         ~World () // A destructor always has the same name as the class but is preceeded with tilde (~).
         {         // A piece of code that is executed every time an object of this class is destroyed.
                   // Never takes any arguments.
             std::cout << "Good bye!" << std::endl; // ----> prints "Good bye!"
         }
  }; // A class is delimited by a semi-colon.

// Object definition --> This is the central line of the program.
  World TheWorld; // --> "TheWorld" is an instance of the class "World".
             // |
             // ----> A statement is delimited by a semi-colon (;).
// Main routine
  void main ()
  {
      return 0;
  }
    • Global scope is everything outside of curly braces.
    • Top-down approach is the correct way of analyzing a program and creating it.

    Program flow:

    1. All global objects are constructed. The constructer executes it's instructions.
    2. The main routine is executed.
    3. All global objects are destroyed. The destructor is executed.
    • Building means compiling and linking.

    Summary: We have introduced a class and an object. Think of a class as a blueprint for an object. It describes the object's behavior - in our case, what happens when the object is constructed and destroyed. Once you have a blue print, you can create objects that are based on it.

    Note: One could think of main as the constructor of the global Main object. This is how truly object oriented languages work.

  2. Local Scope
    • Scope of main
    • Passing arguments to constructors
    • Interger type
    • Private data members
    • Initialization
    • Embedded local scopes
    • The for loop
    • Global scope is activated (ready to use, all objects constructed) before main is executed, and deactivated (all objects destroyed) after main is exited.
    • Local scopes are delimited by braces (not always)
      • Activated when the flow of the program enters it, and deactivated when it leaves it.
      • Objects in local scope are constructed whenever their definition is encountered, and destroyed when the scope is exited.
        • Such objects are called automatic (because they are automatically created and destroyed), or stack objects (because they occupy memory that is allocated on the program's stack).
    #include <iostream>

class World
{
  public:
  World (int i)
  {
    std::cout << "Hello from " << i << "." << std::endl;
  }

  ~World ()
  {
    std::cout << "Good bye." << std::cout;
  }
};

World TheWorld (1);

int main ()
  {
    World myWorld (2);
    std::cout << "Hello from main!" << std::endl;
    return 0;
  }
    • std::cout accepts strings of characters and prints them as string and it accepts integers and prints them as decimal numbers.
    • You can chain arguments to std::cout one after another.
    • An object is something that has identity.
    • If possible, stick to countable nouns in naming classes.
    #include <iostream>

class World
{
  public:
  World (int id)
    : _identifier (id)
  {
    std::cout << "Hello from " << _identifier << "." << std::endl;
  }

  ~World ()
  {
    std::cout << "Good bye from " << _identifier << "." << std::endl;
  }

  private:
    const int _identifier;
};

World TheWorld (1);

int main ()
  {
    World myWorld (2);
    for (int i = 3; i < 6; ++i)
      {
        World aWorld (i);
      }
    World oneMoreWorld (6);

    return 0;
  }

    • The braces delimiting the scope may be omitted if the body of the loop consists of a single statement. 

      for (int i = 3; i < 6; ++i)
  World aWorld (i);
    • The scope of a variable defined in the head of a loop (or the if statement) is that of the loop itself.
    • Variables defined in the body of the loop are re-initialized on every iteration.
    • The ones defined in the head are initialized once, at the loop entry. However, once you exit the loop, neither are accessible.
    • The compiler will make sure that nobody, except the class itself, has access to private members.
    • The keyword const means that nobody can change the value of _identifier during the lifetime of the object.
    • The preamble to the constructor is the little window when we can (and have to) initialize a const data member.

    Summary: Objects may have their own memory, or state. The state data members are preferably kept private. They are best initialized in the preamble to the constructor, so that the object is in a consistent state immediately after its birth. The "for loop" creates its own scope. This scope is entered an exited during every iteration. In fact, a matched pair of braces can be put anywhere within another scope, just to form a sub-scope.
